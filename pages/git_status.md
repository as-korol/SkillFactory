# Как работает команда `git status`
---

Команда git status используется для получения информации о состоянии вашего репозитория Git. В этой странице мы рассмотрим, как работает эта команда и что она показывает.

## Что показывает git status

Когда вы выполняете команду git status, Git предоставляет информацию о:

- Ветке, на которой вы находитесь.
- Изменениях в вашем рабочем дереве (в файлах, которые вы изменили или добавили).
- Изменениях, которые ещё не добавлены (unstaged changes).
- Изменениях, которые были добавлены (changes to be committed).
- Отслеживаемых и неотслеживаемых файлах.

## Основные состояния

1. On branch <branch name>: Эта строка показывает, на какой ветке вы находитесь. 

2. Changes not staged for commit: В этом разделе отображаются изменения в файлах, которые сделаны и еще не прошли этап git add. Они не войдут в следующий коммит, пока вы не выполните git add для этих файлов.

3. Changes to be committed: В этом разделе отображаются изменения, которые вы добавили с помощью git add и ожидают коммита. Это включает изменения, которые будут внесены в следующий коммит.

4. Untracked files: Здесь перечислены файлы, которые не отслеживаются Git. Это новые файлы, которые еще не были добавлены с помощью git add.

## Дополнительные команды

- ```git status -s``` или ```git status --short``` Эта команда отображает более краткую версию вывода git status, показывая только основную информацию о файлах и их статусе.

- ```git status -v``` или ```git status --verbose:``` Эта команда выводит более подробную информацию о файлах и их изменениях.

## Интерпретация статусов

- ```M```: Файл был изменен и не добавлен (unstaged changes).

- ```A```: Файл был добавлен (changes to be committed).

- ```D```: Файл был удален.


## Заключение

Выполнение команды git status позволяет вам получить обзор текущего состояния вашего репозитория. Она помогает вам отслеживать и контролировать ваши изменения и решать, какие файлы должны быть включены в следующий коммит.
