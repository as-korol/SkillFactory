# Как работает команда `git init`
----

Команда git init используется для инициализации нового репозитория Git в выбранной папке. В этой странице мы рассмотрим, как работает эта команда и что происходит во время инициализации.

---

## Шаги инициализации


```
git init
```


Git создаст новую папку .git, которая будет содержать все необходимые файлы и папки для работы с Git.

Инициализация создаст пустой репозиторий без каких-либо файлов. Вам нужно будет добавить файлы с помощью команды git add и выполнить коммит с использованием команды git commit.

После выполнения команды git init, Git создает следующие файлы и папки:

- .git/ - Основная папка репозитория, содержащая все данные Git.
- .git/config - Файл конфигурации репозитория, где хранятся настройки.
- .git/HEAD - Файл, указывающий на текущую ветку.
- .git/hooks/ - Папка, содержащая скрипты хуков, которые выполняются при определенных событиях, таких как коммит или получение изменений из удаленного репозитория.
- .git/objects/ - Папка, содержащая все объекты Git (коммиты, ветки, теги, изменения и т. д.).
- .git/refs/ - Папка, содержащая ссылки на объекты (ветки и теги).

## Изменения в файловой системе

После инициализации репозитория Git будет следить за изменениями файлов в выбранной папке. В каждом коммите сохраняется состояние всех файлов.

- Когда вы вносите изменения в файлы, Git отслеживает эти изменения.
- Когда вы добавляете файлы с помощью git add, Git отмечает эти файлы для включения в следующий коммит.
- Когда вы выполняете коммит с помощью git commit, Git сохраняет изменения во внутренней базе данных репозитория.

Вот основная информация о работе команды git init. Теперь вы готовы начать использовать Git в своем проекте.
