# Как работает команда `git add [file]`
---

Команда git add [file] используется для добавления изменений определенного файла или файлов в индекс Git (также известный как "staging area"). В этой странице мы рассмотрим, как работает эта команда и что происходит при добавлении файлов.

## Что происходит при выполнении git add

Когда вы выполняете команду `git add [file]`, происходят следующие действия:

1. Файлы, указанные в команде `git add`, копируются из вашего рабочего каталога `(working directory)` в индекс Git.

2. Индекс Git содержит снимок состояния файлов, которые будут включены в следующий коммит. Когда вы фиксируете коммит, Git сохраняет состояние файлов из индекса.

3. Если вы вносите изменения в файл, который уже был добавлен с помощью `git add`, при повторном использовании этой команды, Git обновит снимок файла в индексе.

## Добавление файлов

Вы можете добавить определенный файл, указав его имя после команды `git add`. Например:

``` 
git add myfile.txt 

```


Вы также можете добавить несколько файлов, перечисляя их имена через пробел:

``` 
git add file1.txt file2.txt 

```


Если вы хотите добавить все измененные файлы в одной команде, вы можете использовать символ точки (.):

```
$ git add .

```


Это добавит все измененные файлы в вашем рабочем каталоге.

## Использование паттернов

Команда `git add` также поддерживает использование паттернов для добавления файлов. Например:

``` 
$ git add *.txt 

``` 


Это добавит все файлы с расширением `.txt` в текущем каталоге.

## Игнорирование файлов

Git также позволяет вам создать файл `.gitignore`, в котором вы можете указать шаблоны и имена файлов/папок, которые Git должен игнорировать при добавлении. Создайте файл `.gitignore` в корневой папке репозитория и добавьте шаблоны для игнорируемых файлов.
